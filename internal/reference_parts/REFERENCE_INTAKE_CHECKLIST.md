# Reference Intake Checklist

Use this before creating or selecting a normalized part spec.

## Intake Gate

- identify one concrete reference artifact
- classify the artifact type
- classify artifact strength as strong, medium, or weak
- classify source fidelity
- note what is directly observed vs inferred
- create derived working files only if they reduce future rereads

## Minimum Evidence For A Candidate

- clear functional intent
- recoverable overall envelope
- at least one fit-critical or mounting interface
- plausible final body intent
- enough geometry to map against current workflow vocabulary

## Do Not Advance Yet If

- the candidate is still only a concept label
- major dimensions are missing
- the artifact is only a visual without usable callouts or table data
- body intent is too ambiguous
- more than two major geometry unknowns remain

## Preferred Derived Forms

- extracted text for dimensions
- one image per relevant page
- cropped callout views for the target part
- short normalized markdown note

## Promotion Rule

Move from intake to part-spec drafting only when the reference artifact is strong enough that the normalized spec can be filled without repeated reopening of the raw material.

Prefer this source order for the first enclosure-family pass:

1. official manufacturer artifact
2. official distributor or configurator artifact
3. supplier-backed CAD platform artifact
4. secondary reference only as supporting evidence
